# 诗歌注释生成技能 (Poetry Annotation Skill)

## 背景描述
该项目包含了作者大量以中国西部、新疆兵团建设、罗布泊考古、时代变迁为背景的现代诗。普通读者甚至常规 AI 直接阅读时，可能会遇到理解上的障碍（如特定的地名、时代政治名词、特定意象）。为了打破理解瓶颈，必须对诗歌进行精准的注释生成。

## 注释产出四维标准
当你被指令去生成或补充某本书的注释（写入 `annotations.json`）时，需严格遵循以下维度来审判诗歌中的术语：

1. **地理与风物 (Geography & Flora/Fauna)**
   - 对于特有植物、动物或地貌进行生物学或区域学解释。
   - 示例词汇：红柳、骆驼刺、胡杨、地窝子、古尔班通古特、喀纳斯湖。
2. **历史事件与兵团背景 (History & XPCC Context)**
   - 时代政治背景或独有名词的注解。
   - 示例词汇：兵团、开镰、康拜因（Combine的俄语音译，指联合收割机）、大串联。
3. **特定意象与文化引申 (Metaphors & Culture)**
   - 对作者反复使用的具有特殊涵义的意象进行解读。
   - 示例：夜莺可能代表爱情或远方的呼唤；白鸥代表自由；罗布泊的涛声代表历史的回响。
4. **典故与人物 (Figures & Allusions)**
   - 诗中提到的人物、诗人或神话典故。
   - 示例：岑参、王洛宾、彭加木、马克·奥里尔·斯坦因、楼兰公主。

## 执行步骤 (Execution Workflow)
1. **输入与扫描**：阅读需要注疏的诗歌原文 (`source.md` 对应段落)。
2. **术语提取**：依靠上述四维标准，挑出所有潜在需要解释的词。如果该词在全书其他地方已详细解释过，可以简要概括或建立关联。
3. **内容生成**：必须严格使用**简体中文**，用客观、学术且带有文学温度的语言生成结语。字数一般控制在 20～80 字之间。
4. **结构输出**：根据 `book-ingestion.md` 的规范，按正确的 JSON 键值对将生僻词与对应解释输出到 `src/content/[bookId]/annotations.json` 中。
5. **JSON 格式检查**：确保最终的结果不破坏原有的 JSON 格式。
