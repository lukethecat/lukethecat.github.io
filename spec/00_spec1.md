# Spec 00: Initial Setup and Guidelines

## Overview
This specification file serves as the initial setup and outlines the testing and development guidelines for the project.

## Guidelines
- Write good, efficient, fast tests with good coverage.
- Use best practices, write efficient, simplified code, and avoid anti-patterns.
- Always reference documentation for any code, dependencies, or libraries used.
- All communications, error analysis, and logic explanations must be in Simplified Chinese (简体中文).

## Testing
- Use `agent-browser` with dogfooding before requesting user testing.
