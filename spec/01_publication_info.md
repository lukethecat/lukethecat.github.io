# Spec 01: 版权与附记信息解析异常复盘 (Publication Info Parsing Issue)

## 1. 现象描述 (Issue)
在 `heiyingsuxiajuan1998`（黑罂粟·下卷）中，发现书籍最后的出版信息（如“封面题签沈 鹏”等）被错误地一并解析、并入到了该卷最后一首诗歌的正文内容里。用户反馈在多个书籍中也可能存在类似的情况。

## 2. 根因分析 (Root Cause)
通过排查 `scripts/process-books.ts` 的解析逻辑和四本书的 `source.md`，确认了以下原因：

1. **边界判定机制**：`process-books.ts` 的第二遍（Pass 2）解析逻辑依赖于 `## ` 二级标题（或者 `[pX]` 页码标记）来终止一首诗的采集。如果遇到了非章节的二级标题（如 `## 出版信息`、`## 后记`、`## 跋...`），解析器会正确地 `break` 并停止将后续文本吸纳进最后一首诗里。
2. **缺失边界符**：在早期的 `heiyingsuxiajuan1998/source.md` 末尾，出版信息前并没有放置任何 `## ` 级标题。这导致解析器无法察觉诗歌已经结束，从而把最后所有的版权文字都加入到了最后一首诗的 `lines` 数组中。
3. **数据未同步缓存**：虽然在 3月20日 已经为 `heiyingsuxiajuan1998/source.md` 补上了 `## 出版信息` 的字样，但并未重新执行 `process-books.ts` 进行编译。前端仍读取了 3月16日 的陈旧 `book.json` 数据，导致界面上依然显示那些无关信息。

其他书籍情况：
- **`zhungaer1984`**：末尾包含沈阳辽宁协作的跋文，但在其上方正确使用了 `## 跋：从大漠升起袅袅的笛声——读《准噶尔诗草》`，解析器成功在此处中断，**未发生错误吸纳**。
- **`hanxuema1995`**：末尾有 `## 出版信息` 作为屏障，**未发生错误吸纳**。
- **`heiyingsushangjuan1998`**：出版本文与其他元数据独立存放于单独的 `.md` 中，不存在于 `source.md` 末尾，**未发生错误吸纳**。

## 3. 修复措施 (Fixes)
1. 确认了所有的书籍结尾附记均具备正确的 `## ` 分隔。
2. 重新运行了 `npm run process:books` 强制刷新所有 `book.json` 数据。
3. 抽查了 `src/content/heiyingsuxiajuan1998/book.json`，相关出版信息已被成功剔除，不再属于最后一首诗的内容。
4. 运行了 `npm run test:data` 回归测试，检查通过，书籍结构无异常。

## 4. 改进与防范规范 (Prevention SOP)
为了在将来增加新书时彻底切断该类 Bug 出现的概率，需遵守以下 SOP 原则：
1. **物理隔离规范**：所有不属于诗歌正文的卷尾信息（包含版权页、定价、扉页题签、跋文、后记等），**必须且只能**被放置在一个单独的二级标题下（例如：`## 后记`、`## 出版信息`、`## 附录`）。
2. **重编译动作**：一旦对书籍目录、诗歌正文或结尾附加内容（即便是不展示在前端的文字）做了修改，必须手动触发构建命令 `npm run process:books` 重新生成解析数据结构，并通过 `npm run test:data` 进行校验，严禁直接上传未解析的 `source.md`。
